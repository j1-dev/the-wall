## Decisions made (see CONTEXT.md + ADRs for full detail)

- Single global Board, flat append-only stream of Messages
- Message = text (≤500 chars) + timestamp + Author name
- Author = Adjective+Animal+Number (e.g. `SilentFox42`), persisted in `localStorage`, generated client-side
- No moderation, no deletion, no auth
- WebSocket via API Gateway — used for both sending AND receiving Messages
- DynamoDB Streams → dedicated fan-out Lambda pushes new Messages to all active Connections
- Stale Connections (410 Gone) deleted immediately during fan-out
- Last 50 Messages on load, cursor-based pagination (HTTP API) for older ones
- Single DynamoDB table named `the-wall`:
  - Messages: `PK=MESSAGE`, `SK=<ISO timestamp>#<uuid>`
  - Connections: `PK=CONNECTION#<connectionId>`, `SK=CONNECTION#<connectionId>`, `type=CONNECTION`
  - GSI: `type-index` on `type` attribute (used by fan-out to list all Connections)
  - Streams: `NEW_IMAGE`
- Terraform for infra, LocalStack (Pro, running via docker-compose) for local dev
- Node.js runtime for Lambdas
- S3 + CloudFront for frontend hosting
- `config.js` generated by Terraform `local_file` with API URLs for the frontend
- Each Lambda has its own independent `package.json` (no monorepo)

## Current task — Task 4 (IN PROGRESS)

**WebSocket handler Lambda** at `lambdas/websocket-handler/index.js`.

The user was about to write the routing skeleton — a `switch` on `event.requestContext.routeKey` with three branches: `$connect`, `$disconnect`, `$default`. No DynamoDB logic yet, just `console.log` in each branch.

After the skeleton is reviewed, the full handler should:
- `$connect`: write Connection item to DynamoDB
- `$disconnect`: delete Connection item from DynamoDB  
- `$default`: parse `{ text, author }` from body, validate (text ≤500 chars), write Message item to DynamoDB

Setup for the Lambda:
```bash
cd lambdas/websocket-handler
npm init -y
npm install @aws-sdk/lib-dynamodb @aws-sdk/client-dynamodb uuid
```

DynamoDB endpoint hardcoded as `http://localhost:4566` and table name as `the-wall` for now (env vars come later).
